# Contributing to AI Risk Assessment Framework

Thank you for your interest in contributing! This project welcomes contributions that improve the framework's coverage, accuracy, and usability.

## Ways to Contribute

- **New assessment questions** for emerging AI risks (agentic AI, multimodal, generative AI)
- **Additional worked examples** (open-source LLMs, CV systems, recommendation engines)
- **Translations** of the framework and documentation
- **Bug fixes** in Excel formulas or Python scripts
- **GRC platform integrations** (ServiceNow, Archer, OneTrust)

## Process

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature-name`
3. Make your changes with clear commit messages
4. Open a Pull Request with a description of what you changed and why

## Quality Standards

- Assessment questions must be answerable with evidence (not just opinions)
- Controls must be actionable and implementable
- All framework changes must reference a recognized standard (NIST, ISO, OWASP, etc.)
- Python scripts must run without errors on Python 3.9+

## Reporting Issues

Use GitHub Issues to report bugs, suggest improvements, or ask questions. Please include:
- What you expected to happen
- What actually happened
- Steps to reproduce (for bugs)
